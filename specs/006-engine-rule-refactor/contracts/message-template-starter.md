# MessageTemplate Starter (T030 mechanical extraction)

Generated by `tools/message-template-extract/` against the post-PR-3b
rule catalog plus `marque-engine`. Hand-curated into the closed
`MessageTemplate` enum at T031. Each cluster groups source-site
literals that share placeholder shape after `{...}` normalization.

**Curation rules** (per design doc §3 T031):

1. One closed-enum variant per *semantic* (not per per-rule literal).
If two clusters convey the same diagnostic intent, collapse them.
2. Each variant's `MessageArgs` field set MUST be in the closed
permitted set: `TokenId`, `CategoryId`, `Span`, `Blake3Hash`,
`Confidence`, `FeatureId`. No `String`, no `&str`, no `Vec<u8>`.
3. Each variant carries a `// CAPCO-2016 §X.Y pNN` doc comment when
mapped to a specific marking semantic. Verify the citation against
`crates/capco/docs/CAPCO-2016.md` before committing the variant.
4. Engine-synthetic variants (R001/R002, corrections-map, decoder
heuristics) carry no §-citation.
